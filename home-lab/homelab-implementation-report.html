Após instalar o Proxmox, acessar https://192.168.3.10:8006 do Lenovo. As etapas abaixo referem-se à configuração pós-instalação.
bashstorage-setup.sh — lm-claw (2 discos)
# Identificar discos no lm-claw (bare-metal)
lsblk -f
# nvme0n1 → NVMe 238GB (OS + OpenClaw)
# sdb     → HDD 298GB (openclaw-storage)

# HDD 298GB → /mnt/openclaw-storage
fdisk /dev/sdb  # n, p, 1, enter, enter, w
mkfs.ext4 /dev/sdb1
mkdir -p /mnt/openclaw-storage
mount /dev/sdb1 /mnt/openclaw-storage
echo "/dev/sdb1 /mnt/openclaw-storage ext4 defaults,nofail 0 2" >> /etc/fstab

# Verificar
lsblk -f && df -h
bashnetwork-setup.sh
# /etc/network/interfaces
auto lo
iface lo inet loopback

auto enp2s0
iface enp2s0 inet manual

# Bridge principal para VMs
auto vmbr0
iface vmbr0 inet static
    address 192.168.3.10/24
    gateway 192.168.3.1
    bridge-ports enp2s0
    bridge-stp off
    bridge-fd 0

# /etc/resolv.conf
nameserver 192.168.3.5   # AdGuard
nameserver 8.8.8.8
bashproxmox-hardening.sh
# Desabilitar repositório enterprise
sed -i 's/^deb/#deb/' /etc/apt/sources.list.d/pve-enterprise.list

# Habilitar repositório community
echo "deb http://download.proxmox.com/debian/pve bookworm pve-no-subscription" \
  > /etc/apt/sources.list.d/pve-community.list

# Atualizar e instalar ferramentas
apt update && apt full-upgrade -y
apt install -y htop iotop curl git net-tools samba nfs-kernel-server

# Remover aviso de subscription
sed -i.bak "s/data.status !== 'Active'/false/" \
  /usr/share/javascript/proxmox-widget-toolkit/proxmoxlib.js
systemctl restart pveproxy

bashmigrate-openclaw.sh — migração VM do laptop para Optiplex
# 1. No Proxmox do Laptop — criar backup
vzdump 101 --storage local --mode snapshot --compress zstd

# 2. Transferir para o Optiplex via rede local
scp /var/lib/vz/dump/vzdump-qemu-101-*.vma.zst \
    root@192.168.3.10:/mnt/hdd/dump/

# 3. No Proxmox do Optiplex — restaurar
qmrestore /mnt/hdd/dump/vzdump-qemu-101-*.vma.zst 200 \
    --storage hdd-storage \
    --unique true

# 4. Ajustar recursos da VM restaurada
qm set 200 --memory 8192 --cores 4  # Mais recursos no Optiplex
qm start 200

Acesso Externo — Tailscale
bashtailscale-setup.sh
# Instalar em cada nó Proxmox
curl -fsSL https://tailscale.com/install.sh | sh

# Ativar com subnet routing no INSPIRON (nó secundário)
# (não fazer no Optiplex para evitar conflito com Corosync local)
tailscale up \
  --advertise-routes=192.168.3.0/24 \
  --accept-routes

# Habilitar IP forwarding (necessário)
echo 'net.ipv4.ip_forward = 1' >> /etc/sysctl.conf
sysctl -p

# No Tailscale Admin Console
# → Machines → Inspiron → Edit Route Settings
# → Approve 192.168.3.0/24

Fase 1 — Backup e Decommission do Optiplex Proxmox

Playbook: 09-decommission-optiplex-proxmox.yml

  • Verificar Inspiron standalone: pvecm status
  • Desligar o Optiplex fisicamente
◯ Fase 2
Fase 2 — Instalar Debian 12 no Optiplex (manual)
  • Criar USB bootável com Debian 12 netinstall
  • Instalar Debian no NVMe 238GB — IP estático 192.168.3.10, usuário luismarquitti
  • Autorizar chave SSH: ~/.ssh/id_ed25519.pub
  • Adicionar luismarquitti ao grupo sudo
  • Verificar conectividade: ssh luismarquitti@192.168.3.10
  • Atualizar reserva DHCP do Optiplex no Mercusys: MAC 74:86:7A:FA:8E:C8 → IP .30
◯ Fase 3
Fase 3 — Provisionar OpenClaw Bare-Metal via Ansible

Playbook: 07-openclaw-deploy.yml

  • Rodar playbook: sistema base, NVM, Node.js LTS, Ollama, Tailscale, diretórios
  • Instalar OpenClaw: pnpm install -g openclaw (manual — requer autenticação)
  • Restaurar dados: tar xzf openclaw-data-backup.tar.gz -C /
  • Rodar openclaw configure e renovar token Google
  • Executar openclaw gateway install --force && openclaw gateway start
  • Executar openclaw doctor --fix e validar
  • Pull dos modelos Ollama: gemma4:e2b, gemma3:4b, qwen3:8b
  • Montar HDD 298GB: /mnt/openclaw-storage (ext4)
  • Autenticar Tailscale: tailscale up --advertise-routes=192.168.3.0/24
◯ Fase 4
Fase 4 — Criar AdGuard e Nginx no pve-inspiron

Playbook: 08-inspiron-migrate-services.yml

  • Criar LXC 100 (adguard) no Inspiron: 192.168.3.5, 512MB
  • Instalar AdGuard Home — configurar DNS Rewrites para todos os serviços
  • Atualizar DNS primário no Mercusys para 192.168.3.5
  • Criar LXC 102 (nginx-proxy) no Inspiron: 192.168.3.7, 1GB
  • Configurar proxy hosts: lm-claw (.30), homeassistant (.22), pve-inspiron (.50), uptime (.6)
  • Testar resolução DNS da rede e acessos por hostname .lan
  • Verificar RAM do Inspiron: free -h — alvo: <7GB usados de 8GB
