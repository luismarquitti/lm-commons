# Recomendação Arquitetural: Nó de Mídia (Plex/Jellyfin)

Para a expansão do cluster Proxmox com o novo nó (192.168.3.51), a recomendação foca em performance de I/O e centralização de dados.

## 1. Estratégia de Discos
- **Disco do Sistema (SSD):** Instale o Proxmox VE em um SSD local. Reserve uma partição ou use o LVM-Thin local para o disco raiz do LXC/VM que rodará o Plex/Jellyfin.
- **Cache de Transcode:** Utilize `/dev/shm` (RAM) para o diretório de transcode temporário. Se a RAM for limitada, use um diretório no SSD local.
- **Repositório de Mídia (NFS):** Não armazene os arquivos de mídia (filmes/séries) localmente no novo nó se eles já existirem no disco de 1TB do `pve-optiplex`. Utilize o compartilhamento NFS já configurado no ecossistema para montar esses dados.

## 2. Virtualização (LXC vs VM)
- **Recomendação:** Utilize um Container LXC (base Debian/Ubuntu).
- **Vantagem:** Facilita o passthrough de hardware (iGPU Intel QuickSync ou GPU NVIDIA) para transcodificação por hardware, essencial para o Plex/Jellyfin sem sobrecarregar a CPU.

## 3. Rede
- **VLAN:** Se possuir segmentação, coloque o serviço de mídia na VLAN de serviços, mas garanta que ele possa acessar o IP do `pve-optiplex` (192.168.3.10) para o NFS.
