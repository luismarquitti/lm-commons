# ADR-008: Bi-directional Traceability via YAML Frontmatter

## Status

Accepted

## Date

2026-05-19

## Context

We need to maintain a strong link between the local specification files (`task_NN.md`) and the external task tracker (Linear) to ensure traceability.

## Decision

We will store the Linear Issue ID in the YAML frontmatter of each `task_NN.md` file using the key `linear_issue_id`.
Example:
```yaml
---
status: pending
title: "feat(billing): add PIX support"
type: backend
complexity: medium
dependencies: []
linear_issue_id: "CLE-123"
---
```

## Alternatives Considered

### Alternative 1: Mapping file
- **Description**: `.compozy/tasks/<name>/_linear_map.json`.
- **Pros**: Keeps task files "clean".
- **Cons**: Fragmented context; easier to get out of sync.
- **Why rejected**: User preferred direct inclusion in the task files.

## Consequences

### Positive
- Each task file is self-describing and knows its external reference.
- Agents can easily find the Linear issue associated with a task they are executing.

### Negative
- Local files are modified by the sync tool.

### Risks
- Manual editing of the ID might break synchronization.

## Implementation Notes

The `sync-compozy-tasks` tool in the `linear` skill must be able to write these IDs back to the files.
