# ADR-004: Adopt an existing open-source dashboard in Phase 3

## Status

Accepted

## Date

2026-05-19

## Context

The target end state includes a command center to visualize and manage running agents. This
could be built from scratch or adopted from existing tools. As of 2026 several mature
open-source options exist — PI Dashboard, Mission Control, ClawPort — alongside the native
Claude Code Agent View.

## Decision

Adopt an existing open-source dashboard (PI Dashboard, Mission Control, or ClawPort) as the
command center, introduced in Phase 3. Do not build a custom dashboard. The specific tool is
selected when Phase 3 is scoped.

## Alternatives Considered

### Alternative 1: Build a custom dashboard

- **Description**: Develop a bespoke command center.
- **Pros**: Exact fit to the workflow.
- **Cons**: Significant build and maintenance cost for a solo developer; duplicates mature
  open-source tools.
- **Why rejected**: The cost is not justified when proven tools exist.

### Alternative 2: CLI-only (native Claude Code Agent View)

- **Description**: Rely solely on CLI tooling, with no web dashboard.
- **Pros**: Zero additional infrastructure.
- **Cons**: Weaker at-a-glance visibility across many concurrent agents and projects.
- **Why rejected**: A command center is an explicit goal; CLI views remain a useful complement
  but not the primary surface.

## Consequences

### Positive

- No build or maintenance burden for the dashboard.
- Benefits from a maintained, community-supported tool.

### Negative

- Constrained to the adopted tool's feature set and conventions.

### Risks

- The chosen tool may be abandoned upstream. Mitigation: prefer a tool with active maintenance
  and self-hostable, dependency-light design; defer the final pick to Phase 3.

## Implementation Notes

The dashboard is a Phase 3 deliverable; tool selection is an open question until then.
