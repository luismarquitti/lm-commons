# ADR-007: Centralized Workflow Playbook in AI Commons

## Status

Accepted

## Date

2026-05-19

## Context

The "Agentic Development Workflow" needs a single source of truth for its definition to ensure consistency across multiple projects (IntelliFinance, ClinicCare, etc.).

## Decision

The primary Workflow Playbook will be centralized in the `ai-commons` repository at:
`.ai-commons/instructions/workflow/PLAYBOOK.md`

Repositories using the workflow may contain a local `AGENTS.md` (or similar) that points to this central playbook to ensure agents have access to the rules.

## Alternatives Considered

### Alternative 1: Per-repository documentation
- **Description**: `docs/WORKFLOW.md` in each repo.
- **Pros**: Local and self-contained.
- **Cons**: High maintenance burden to sync changes; drift between projects.
- **Why rejected**: User preferred a Single Source of Truth (SSOT).

## Consequences

### Positive
- One place to update the workflow.
- Consistent behavior across all "Universal AI Commons" compatible projects.

### Negative
- Requires agents to have access to `ai-commons` or for it to be stowed into the local environment.

### Risks
- If `ai-commons` is not present, agents might lack guidance.

## Implementation Notes

The playbook should be readable by all agents and clearly define roles (Orchestrator, Developer, Reviewer).
