# ADR-001: Phased rollout with a human-in-the-loop MVP

## Status

Accepted

## Date

2026-05-19

## Context

The Agentic Development Workflow initiative is broad: a spec pipeline, Linear-based task
management, isolated parallel work, an agent review loop, autonomous orchestration on OpenClaw,
and a command-center dashboard. Delivering all of it at once risks high overhead, unvalidated
assumptions, and scope creep before the core workflow has proven itself.

## Decision

Deliver the workflow in three phases. The MVP (Phase 1) is the local human+agent SDD workflow
with the developer triggering every transition manually. Event-driven automation and cloud
execution come in Phase 2. Fully autonomous orchestration on OpenClaw and the command-center
dashboard come in Phase 3. Each phase has explicit success criteria that gate the next.

## Alternatives Considered

### Alternative 1: Full autonomy from the start

- **Description**: Build the autonomous OpenClaw orchestration and event-driven agent reactions
  in the first release.
- **Pros**: Reaches the target end state fastest if everything works.
- **Cons**: Large unvalidated surface; failures are hard to diagnose without a proven manual
  baseline; high overhead before any value is delivered.
- **Why rejected**: The core workflow must be proven with a human in the loop before automation
  is layered on.

### Alternative 2: Dashboard-first

- **Description**: Build the command center before the workflow itself.
- **Pros**: Early visibility into agent activity.
- **Cons**: There is little to visualize until the workflow exists; inverts the value order.
- **Why rejected**: Visibility is worthless without an underlying workflow to observe.

## Consequences

### Positive

- The core workflow is validated cheaply before automation is built on top of it.
- Each phase delivers usable value and has a clear gate.
- Scope creep is contained by per-phase success criteria.

### Negative

- The full autonomous end state is reached later than a single-shot build.

### Risks

- The developer may revert to ad-hoc habits during the manual MVP. Mitigation: measure
  repeatability explicitly and keep the MVP lean.

## Implementation Notes

Phase boundaries and success criteria are defined in the PRD "Phased Rollout Plan" section.
