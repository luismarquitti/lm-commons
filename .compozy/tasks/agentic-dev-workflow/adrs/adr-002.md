# ADR-002: Compozy `cy-*` skills as the SDD framework

## Status

Accepted

## Date

2026-05-19

## Context

The workflow needs a Spec-Driven Development framework to take a feature idea through PRD,
TechSpec, and task breakdown. Two mature options exist in the user's skill ecosystem: the
Compozy `cy-*` skills and the `tlc-spec-driven` skill. The choice shapes the structure of every
spec artifact the workflow produces.

## Decision

Anchor the spec pipeline on the Compozy `cy-*` skills (`cy-create-prd`, `cy-create-techspec`,
`cy-create-tasks`, `cy-execute-task`, `cy-review-round`, `cy-fix-reviews`). Spec artifacts are
versioned under `.compozy/tasks/` in each repository.

## Alternatives Considered

### Alternative 1: `tlc-spec-driven`

- **Description**: Use the `tlc-spec-driven` skill with its four adaptive phases — Specify,
  Design, Tasks, Execute.
- **Pros**: Auto-sizes depth by complexity; single integrated skill.
- **Cons**: Less granular per-stage control; the Compozy skill set already covers review
  rounds and review-fix loops, which align directly with this workflow's review gate.
- **Why rejected**: The Compozy `cy-*` set maps more cleanly onto the workflow's distinct
  stages, including the PR review loop.

### Alternative 2: Keep both options open

- **Description**: Defer the framework choice and record it as a pending decision.
- **Pros**: Maximum flexibility.
- **Cons**: An undefined artifact structure blocks the workflow from being documented and
  followed consistently.
- **Why rejected**: The workflow needs one defined artifact structure to be repeatable.

## Consequences

### Positive

- One consistent artifact structure across the spec pipeline.
- The review-round and review-fix skills align directly with the Agent Review Gate.
- Artifacts live in `.compozy/`, which both project repositories already contain.

### Negative

- The workflow is coupled to the Compozy skill set's conventions and evolution.

### Risks

- Tooling churn in Compozy. Mitigation: the Workflow Playbook depends on stable roles and
  conventions, not on internal tool behavior.

## Implementation Notes

Both `intellifinance` and `ClinicCare` already contain a `.compozy/` directory.
