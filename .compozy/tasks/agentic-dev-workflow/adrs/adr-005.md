# ADR-005: Extended Linear Skill for Task Synchronization

## Status

Accepted

## Date

2026-05-19

## Context

The "Agentic Development Workflow" requires a bridge between Compozy-generated task files (`task_NN.md`) and Linear issues. We need a way to register tasks in Linear and maintain synchronization.

## Decision

We will extend the existing `linear` skill in `ai-commons` to include a `sync-compozy-tasks` command/tool. This tool will:
1. Parse the `.compozy/tasks/` directory of a project.
2. Create corresponding issues in Linear if they don't exist.
3. Update existing issues if the task file content changes (optional for MVP).
4. Write the resulting Linear Issue ID back to the task file's frontmatter.

## Alternatives Considered

### Alternative 1: New CLI tool in `ai-commons/scripts`
- **Description**: A standalone script (e.g., `lms sync-tasks`).
- **Pros**: Decoupled from the skill system.
- **Cons**: Less discoverable for agents; duplicates logic for Linear API interaction.
- **Why rejected**: User preferred extending the existing skill to leverage current tools and configuration.

### Alternative 2: Background agent/service
- **Description**: A service watching for file changes.
- **Pros**: Automatic synchronization.
- **Cons**: Complex to implement and maintain; might trigger sync at inappropriate times (e.g., while editing).
- **Why rejected**: Overly complex for Phase 1.

## Consequences

### Positive
- Centralizes Linear logic in one skill.
- Agents can use the same skill they already use for other Linear tasks.
- Leverages existing Linear MCP server configuration.

### Negative
- Increases the complexity of the `linear` skill.

### Risks
- Dependency on Linear MCP server availability.

## Implementation Notes

The tool should be part of the `linear` skill's `SKILL.md` or as a helper script referenced by it.
