# ADR-003: Pilot on IntelliFinance; repo unification out of scope

## Status

Accepted

## Date

2026-05-19

## Context

Two decisions bound the scope of this initiative. First, the workflow could launch on one
project or on both (`intellifinance` and `ClinicCare`) at once. Second, the user also wants to
consolidate `.ai-commons` and `home-lab` into a single versioned GitHub repository — related
work, but distinct from the development workflow.

## Decision

Pilot the workflow on a single project, IntelliFinance, before extending it to ClinicCare in
Phase 2. Exclude the `.ai-commons` + `home-lab` repository consolidation from this initiative;
treat it as a separate Phase 0 infrastructure prerequisite tracked independently.

## Alternatives Considered

### Alternative 1: Launch on both projects at once

- **Description**: Apply the MVP workflow to IntelliFinance and ClinicCare simultaneously.
- **Pros**: Both projects benefit immediately.
- **Cons**: Doubles the surface for discovering workflow problems; ClinicCare's LGPD-sensitive
  data adds constraints better handled after the workflow is proven.
- **Why rejected**: A single pilot validates the workflow at lower risk.

### Alternative 2: Include repo unification in this PRD

- **Description**: Make the `.ai-commons` + `home-lab` consolidation a feature of this PRD.
- **Pros**: One combined plan.
- **Cons**: Repo consolidation is an infrastructure migration with different actors and risks;
  bundling it dilutes the workflow's focus.
- **Why rejected**: Different problem domain; it belongs in its own effort.

## Consequences

### Positive

- The workflow is validated on one project at lower risk before broadening.
- This PRD stays focused on the development workflow.
- ClinicCare's LGPD constraints are addressed once the workflow is already proven.

### Negative

- ClinicCare does not benefit from the workflow until Phase 2.
- The repo-unification goal is tracked outside this PRD.

### Risks

- The Phase 0 repo unification may be neglected. Mitigation: it is named explicitly as a
  prerequisite in the PRD Non-Goals.

## Implementation Notes

IntelliFinance already contains `.compozy/tasks/` and is on GitHub, making it ready for the
pilot.
