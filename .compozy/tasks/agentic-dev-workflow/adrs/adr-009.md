# ADR-009: Slash Command Trigger for PR Reviews

## Status

Accepted

## Date

2026-05-19

## Context

In the MVP (Phase 1), we want a simple way to trigger the Reviewer agent on a Pull Request without full automation or background listeners.

## Decision

The Reviewer agent will be triggered by a human developer using a slash command like `/review` in the PR comment or the agent's chat interface.

## Alternatives Considered

### Alternative 1: Tool call by developer agent
- **Description**: `cy-execute-task` automatically triggers `cy-review-round`.
- **Pros**: Reduces human steps.
- **Cons**: Might trigger too early; agent might review its own work in a loop without human oversight in the MVP.
- **Why rejected**: User preferred manual triggering for better control in Phase 1.

## Consequences

### Positive
- Human-in-the-loop control.
- Simple to implement (no webhooks or watchers needed).

### Negative
- Requires a manual step for every review cycle.

### Risks
- Developer might forget to trigger the review.

## Implementation Notes

The command should be documented in the Playbook.
