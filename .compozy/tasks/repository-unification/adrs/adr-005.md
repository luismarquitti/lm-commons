# ADR-005: Adopt GNU Stow as the artifact distribution mechanism

## Status

Accepted

## Date

2026-05-19

## Context

The SSOT artifacts in `ai-commons/` must reach the tool directories `~/.claude`, `~/.agents`,
and `~/.gemini`. Codebase exploration found that GNU Stow is not installed, the
`stow-packages/base-agent/` tree exists but is empty and unused, and the symlinks currently
present in the tool directories were created manually with no script or documentation. The PRD
constrains distribution to GNU Stow.

## Decision

Adopt GNU Stow as the distribution mechanism. Build real Stow packages under
`ai-commons/stow-packages/` (one per tool target), install Stow as a declared dependency, and
replace the manual symlinks with Stow-managed symlinks. An idempotent bootstrap script installs
Stow if missing and applies the packages.

## Alternatives Considered

### Alternative 1: Formalize manual symlinks with an install script

- **Description**: An idempotent script creates and repairs the symlinks directly, without
  Stow.
- **Pros**: No new tool dependency.
- **Cons**: Reimplements, less robustly, what Stow already does (conflict detection, clean
  removal, dry-run); contradicts the PRD constraint.
- **Why rejected**: Stow is purpose-built and lightweight; a hand-rolled script is inferior.

### Alternative 2: Hybrid — Stow for new packages, keep existing manual symlinks

- **Description**: Use Stow only for newly added packages.
- **Pros**: Less immediate change.
- **Cons**: Two distribution mechanisms coexist, leaving the layout inconsistent and harder to
  reason about.
- **Why rejected**: A single mechanism is clearer and matches the consolidation goal.

## Consequences

### Positive

- One declarative, well-understood distribution mechanism.
- Conflict detection and clean unstow come for free.
- The empty `stow-packages/` scaffold is finally put to use.

### Negative

- Adds GNU Stow as an environment dependency.

### Risks

- Stow refuses to symlink when a real file already exists at the target. Mitigation: the
  bootstrap script removes the pre-existing manual symlinks before stowing and runs a Stow
  dry-run first.

## Implementation Notes

A `.stow-local-ignore` file keeps repository metadata out of Stow operations. Stow packages
mirror the `$HOME`-relative target structure and reference SSOT content under `ai-commons/`.
