# ADR-002: Preserve `home-lab` commit history via subtree import

## Status

Accepted

## Date

2026-05-19

## Context

`home-lab/` is already a GitHub repository (`luismarquitti/home-lab`) with 135 tracked files
and an established commit history of infrastructure decisions. When `home-lab` moves into the
unified monorepo (ADR-001), that history can either be carried over or discarded in favor of a
fresh start.

## Decision

Import `home-lab`'s existing commit history into the unified repository, placed under the
`home-lab/` directory, so that past infrastructure decisions remain traceable. The history is
brought in via a subtree-style import.

## Alternatives Considered

### Alternative 1: Start fresh with no imported history

- **Description**: Copy `home-lab`'s current files into the unified repository as a single
  initial commit; keep the old repository only as an archived reference.
- **Pros**: Simpler one-time import.
- **Cons**: The rationale behind past infrastructure changes becomes harder to recover; `git
  blame` and history-based investigation are lost for `home-lab` content.
- **Why rejected**: The infrastructure history has ongoing diagnostic value.

## Consequences

### Positive

- Past infrastructure decisions stay traceable via `git log` and `git blame`.
- The unified repository is a complete record, not a snapshot.

### Negative

- The one-time import is more involved than a plain file copy.

### Risks

- History could be lost or corrupted during the import. Mitigation: verify the commit history
  after import; the old `home-lab` repository remains intact as a fallback until archived.

## Implementation Notes

The old `luismarquitti/home-lab` repository is archived (read-only) after the import is
verified, to prevent split-brain commits — see the PRD Phase 2 plan.
