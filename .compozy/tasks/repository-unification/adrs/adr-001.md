# ADR-001: Single monorepo with domain top-level directories

## Status

Accepted

## Date

2026-05-19

## Context

Phase 0 must consolidate two bodies of work into one version-controlled GitHub repository:
`.ai-commons/` (the unversioned SSOT for AI artifacts) and `home-lab/` (Proxmox/Ansible
infrastructure already versioned in its own GitHub repository). The two domains are distinct
but the developer wants a single place to record changes and back up the work. The repository
structure must keep the domains legible while giving one coherent change history.

## Decision

Use a single private GitHub repository — a true monorepo — with `ai-commons/` and `home-lab/`
as top-level directories. The repository has one forward commit history. There are no git
submodules.

## Alternatives Considered

### Alternative 1: Flat repository rooted at `.ai-commons`

- **Description**: Make `.ai-commons` itself the git root and move `home-lab` in as a subfolder.
- **Pros**: Slightly less restructuring of the existing SSOT.
- **Cons**: Nests infrastructure IaC inside the AI-commons folder structure, blurring the two
  domains and making the layout harder to read.
- **Why rejected**: Mixing the domains reduces clarity for no real gain.

### Alternative 2: Meta-repository with git submodules

- **Description**: A parent repository referencing `.ai-commons` and `home-lab` as submodules.
- **Pros**: Each domain stays independently versioned.
- **Cons**: Submodules add friction — commits and pushes at two levels, detached-HEAD pitfalls,
  pinned-commit drift — which conflicts with the goal of one simple place to record changes.
- **Why rejected**: The added workflow friction contradicts the core goal.

## Consequences

### Positive

- One repository and one commit history — a single place to record and back up all changes.
- The two domains stay clearly separated by top-level directory.
- No submodule workflow overhead.

### Negative

- The two domains share one issue/history space even though they evolve independently.

### Risks

- The repository could grow large over time. Mitigation: artifacts are configuration and text,
  not heavy binaries; size is not expected to be a concern.

## Implementation Notes

`home-lab` is already a GitHub repository; its content moves under the `home-lab/` directory of
the unified repository with history preserved (see ADR-002).
