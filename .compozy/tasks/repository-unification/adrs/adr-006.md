# ADR-006: Import `home-lab` history via `git subtree add`

## Status

Accepted

## Date

2026-05-19

## Context

ADR-002 requires preserving `home-lab`'s commit history when it moves into the `lm-commons`
monorepo under the `home-lab/` directory. `home-lab` is a git repository with 11 commits, a
`main` branch, and a `feat/openclaw-bare-metal` feature branch currently checked out with
uncommitted changes. A concrete import mechanism is needed.

## Decision

Import `home-lab` using `git subtree add --prefix=home-lab`. Before the import, finalize the
`feat/openclaw-bare-metal` branch in the old repository (commit and merge it into `main`) so
that the imported `main` is complete. The subtree import brings every `home-lab` commit into
`lm-commons` with paths placed under `home-lab/`.

## Alternatives Considered

### Alternative 1: `git-filter-repo`

- **Description**: Rewrite `home-lab`'s history to move every file under `home-lab/`, then
  merge the rewritten history.
- **Pros**: Produces a history where every commit already shows the `home-lab/` prefix.
- **Cons**: A more elaborate, multi-step process; rewriting history is heavier than the task
  needs for a personal repository.
- **Why rejected**: `git subtree add` achieves history preservation in one command with less
  ceremony.

## Consequences

### Positive

- `home-lab` history is preserved in `lm-commons` with a single command.
- `git log` and `git blame` continue to work for `home-lab/` content.

### Negative

- Subtree merge commits add one synthetic merge point to the unified history.

### Risks

- Uncommitted work on `feat/openclaw-bare-metal` is lost if not finalized first. Mitigation:
  finalizing that branch is an explicit prerequisite step in the build order.

## Implementation Notes

After the import, verify the `home-lab/` commit count matches the source (11 commits). The old
repository stays intact until the count is verified.
