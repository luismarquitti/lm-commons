# ADR-003: Complete artifact migration into the SSOT before unifying

## Status

Accepted

## Date

2026-05-19

## Context

AI artifacts are spread across several unversioned locations: the `.ai-commons/` SSOT plus
`.claude/`, `.agents/`, and `.gemini/`. Some of those scattered artifacts are Stow distribution
targets of `.ai-commons`, but — per the `ai-commons-manager` methodology — not everything has
been migrated into the SSOT yet. The unification can either capture only what is in
`.ai-commons` today, or first complete the migration of stray artifacts.

## Decision

Complete the migration of scattered artifacts from `.claude/`, `.agents/`, and `.gemini/` into
the `.ai-commons` SSOT — deduplicating by keeping the most complete and recent version of each
— before building the unified repository. The repository is therefore born complete.

## Alternatives Considered

### Alternative 1: Unify only the current `.ai-commons` content

- **Description**: Version `.ai-commons` and `home-lab` as they are today; treat the migration
  of stray artifacts as separate later work.
- **Pros**: Faster to a first push.
- **Cons**: The repository would not be the true single source of truth; un-migrated artifacts
  remain unversioned and at risk; a second consolidation pass would be needed later.
- **Why rejected**: It defers the core value — one authoritative, versioned copy of everything.

## Consequences

### Positive

- The unified repository is the genuine single source of truth from day one.
- No un-migrated artifacts are left unversioned and unprotected.
- Deduplication removes ambiguity about which copy is authoritative.

### Negative

- Phase 1 takes longer because migration and deduplication happen before the first push.

### Risks

- Deduplication may keep the wrong version of an artifact. Mitigation: compare before merging,
  keep the most complete/recent, and have the developer review dedup decisions.

## Implementation Notes

The `ai-commons-manager` skill defines the migration and deduplication procedure: analyze,
merge, move, then re-establish Stow symlinks.
