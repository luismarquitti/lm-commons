# ADR-008: Re-point all consumers to the new path; no compatibility symlink

## Status

Accepted

## Date

2026-05-19

## Context

The SSOT content currently lives at `~/.ai-commons` and moves into `~/lm-commons/ai-commons/`.
The existing tool symlinks (`~/.claude/skills`, `~/.agents/skills`, `~/.gemini/skills`) and any
scripts referencing the absolute path `~/.ai-commons` break when the content moves. The
transition can either keep a compatibility symlink at the old path or re-point every consumer.

## Decision

Re-point every consumer to the new location and do not keep a `~/.ai-commons` compatibility
symlink. Tool symlinks are regenerated by GNU Stow against `~/lm-commons/ai-commons/`. Scripts
that referenced the old absolute path are changed to resolve their paths relative to their own
location inside the repository, so they are path-independent. The end state has one canonical
path and no legacy alias.

## Alternatives Considered

### Alternative 1: Keep a `~/.ai-commons` compatibility symlink

- **Description**: Leave `~/.ai-commons` as a symlink to `~/lm-commons/ai-commons` so any
  reference to the old path keeps working.
- **Pros**: Lower immediate risk; nothing referencing the old path breaks.
- **Cons**: Leaves a permanent legacy alias and hides stale references instead of fixing them;
  contradicts the single-source-of-truth goal.
- **Why rejected**: A clean final state with no hidden indirection is worth the one-time effort
  of finding every reference.

## Consequences

### Positive

- One canonical path; no legacy alias or hidden indirection.
- Path-relative scripts become portable and survive future moves.

### Negative

- Every reference to `~/.ai-commons` must be found and updated in one pass.

### Risks

- A missed reference breaks silently after the old path is gone. Mitigation: grep the workspace
  for `.ai-commons` before removing the old directory; the environment verification script
  fails loudly on unresolved symlinks.

## Implementation Notes

Known references to update include `ai-commons/scripts/sync-mcp.py` and the `ai-commons-manager`
skill. The old `~/.ai-commons` directory is removed only after verification passes.
